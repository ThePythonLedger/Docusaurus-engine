🏗️ Project 1: The Ledger of Fate

You have officially conquered the fundamentals of Python! Up to this point, you have learned how to write instructions that execute sequentially. But in the real world, programs aren't just straight lines; they are complex webs of decisions, shifting states, and branching pathways.

The ultimate test of a programmer is not just knowing individual syntax rules, but organizing them into a cohesive, living system. In this project, you will build a state-driven Text-Based Adventure Game that simulates a digital vault. You will design the rules of survival, track player state across branches, and program the logical gates that determine whether a player escapes or meets their doom.

🎯 Learning Outcomes​

By the end of this project, you will be able to:

  • Manage Program State: Initialize and dynamically update mutable variables (int, bool) to track a player's journey.
  • Normalize User Input: Capture and sanitize raw user strings using string methods to prevent logic crashes.
  • Architect Nested Control Flow: Design if/elif/else structures nested inside other conditional blocks to create complex, multi-layered decision trees.
  • Implement Compound Boundary Conditions: Use logical operators (and, or) to evaluate whether a player has met multiple distinct survival criteria.

🧠 Conceptual Overview: State & Control Flow​

Before writing code, we must understand how a game engine tracks reality under the hood.

1. The Concept of "State"​

In programming, state is the stored configuration of a system at a specific moment in time. Think of it as your program's short-term memory. In Python, we represent state using variables.

When a player makes a choice, the program does two things:

  1. It evaluates the choice using logical branches.
  2. It modifies the state (e.g., subtracting from health or switching has_token from False to True).

2. Deep Nesting and Execution Contexts​

When you nest an if statement inside another if statement, you are creating a conditional hierarchy:

if choice1 == "left":
    # --- Context level 1 ---
    if choice2 == "right":
        # --- Context level 2 ---
        print("You found the exit!")

Python evaluates the inner conditional block only if the outer condition evaluates to True. If the outer condition is False, the interpreter completely skips the inner block, saving memory and processing time.

📚 Assignments​

To successfully complete this project, you must read and research the following official documentation. Focus on how these concepts apply to dynamic decision-making.

  • Python Official Docs: More Control Flow Tools
    • Focus Area: Read Section 4.1 (if statements). Pay close attention to how elif acts as a chain and how indentation defines block execution.
  • Real Python: Conditional Statements in Python
    • Focus Area: Study the section on "Nested Conditional Statements" and look at how the nested indentation levels align.
  • Python Official Docs: Common String Methods
    • Focus Area: Search for and study the .lower() method. Understand why returning a new string is necessary (due to string immutability).

❓ Knowledge Checks​

Do not guess! Use the assignment links above to research and answer these questions for yourself before writing your project code.

  1. The Immutability Trap: If you have a variable choice = "LEFT" and you run choice.lower(), what is the value of choice on the very next line? Why?
  2. The Dangling Else: In a nested if/else structure, how does Python determine which if statement an else block belongs to?
  3. Short-Circuit Evaluation: In the expression if health > 0 and has_token:, if health is 0, does Python even check the value of has_token? Why or why not?

🏆 The Ledger Challenge​

Your mission is to complete the game engine below. You are trapped in a digital vault. To escape, you must navigate through the nodes, update the player's state, and handle nested choices.

Requirements:​

  1. Dynamic State: You must track and modify health (integer) and has_token (boolean).
  2. Input Sanitization: Use .lower() on all user inputs to handle capitalization gracefully.
  3. Nested Path: Implement a secondary choice (choice2) inside both the left and right major branches.
  4. Win/Loss Boundary: Implement a final gate check. To win, the player must have a health score greater than 0 and possess the token.

💡 Documentation Hunting Tip: Look closely at the Assignments link for Common String Methods. When a user inputs "Left ", it might fail because of the trailing space. Can you find a string method in the official documentation to "strip" whitespace away?
